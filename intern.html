Summer 2020 - Microsoft Research (Reinforcement Learning)

---

During Summer 2020, I am an independent research developer as part of Microsoft Research's Reinforcement Learning Open Source Fest, for which students complete a four-month research programming project aimed to “solve open source reinforcement learning problems and advance the state-of-the-art research and development alongside the RL community”.

---

The project title is: Parallelized Parsing in C++. The project abstract is: "Modern machines often utilize many threads to achieve good performance. Currently, VW uses a single thread to read in and parse input, and a single thread to learn. The parse thread presents a bottleneck, slowing down VW as a whole. By extracting the input reading into a separate thread and extending the parser to support many threads, VW can better utilize resources, achieve better performance, and have an improved design by separating logical components into independent modules. This project focuses on improving performance and design for the text input format, and also ensures compatibility with the cache input format."

---

For this project I contributed to Vowpal Wabbit (VW), an open source machine learning developed by Microsoft Research. I extracted out the I/O processes and created a separate parallel process for input, and multithreaded the parsing component. The VW user can select how many threads to use for the parsing component using a command-line option.

---

I was 1 of 6 selected students out of a pool of over 200 undergraduate, masters, and PhD applicants (3 undergraduate, 3 graduate students accepted) for the Microsoft RL Open Source Fest.

Summer 2020 - Google

During Summer 2020, I am a STEP software engineering intern for the YouTube Ad Formats - Infeed team. \

---

I proposed and worked on Video Vigilance, which uses machine learning and computer vision techniques from the Google Cloud Vision API and Video Intelligence API to detect unsafe or negative content in video advertisements. I was responsible for the workflow of parsing keyframe images, detecting negative content, and displaying results using html. I came up with the original idea for Video Vigilance, which the YouTube Ads – Formats team approved entirely.

---

For Video Vigilance I also wrote design docs and constructed visuals of the user flows and distribution of tasks. For the back-end, I implemented RESTful Java servlets to post, get and delete blobs from DataStore and Google Cloud Storage. I focused on extensibility and edge-case handling in Java classes calling the Cloud Vision API to extract the effect of keyframe images from the video. For the front-end, I implemented a streamlined, professional user interface using Bootstrap.

---

For the internship, I also wrked on a Personal Portfolio website, implementing a comments section on each page and using Google APIs such as the Google Maps API, Blobstore API, and Google Charts API.

During my summer internship at Amazon, I designed and implemented a full-stack web application to streamline the AmazonAPI governance body’s API approval process. For the project, I programmed in JavaScript, NodeJS, Python, HTML, CSS, and Markdown. The website provides filters and a search bar to sort APIs by ETA, status, alias, or title. Page users can dynamically view and revise submitted APIs and post comments. For the APIs displayed on the website, theere is a description of the API or contract change and a conversation section converted from Markdown to HTML. The conversation section is updated and synced automatically among all users and serves as a place where different engineers can communicate about the requested changes.

---

The website automates user authentication and weekly email notifications using internal Amazon tools as well as AWS resources. For back-end development, I created three APIs and eight AWS Lambda functions, as well as a scheduled task runner and a Markdown-to-HTML converter. I created, queried, and updated a DynamoDB mailing list database using AWS API Gateway and AWS Lambda. For front-end development, I implemented an intuitive, responsive user interface using the Bulma CSS framework.

---

During the summer internship, I wrote >100k lines of code over 12 weeks that were pushed to production.
